# 高松灯看板娘 — 功能说明文档

**日期**: 2026-06-10  
**角色**: 高松灯（小灯 / 咕咕嘎嘎）— 企鹅娘，文静害羞的二次元少女  
**状态**: 已集成

---

## 一、概览

JavaTutor 前端右下角嵌入了一个高松灯看板娘（Live2D 动态企鹅娘角色），作为全局 UI 陪伴插件。启动项目后自动加载，屏幕宽度大于 768px 时显示。

核心能力：眼球/头部跟随鼠标、鼠标悬浮 UI 元素出现萌系对话气泡（兼具功能介绍）、鼠标在不同功能区域时表情自动变化、点击身体触发动作+彩蛋、7 个交互工具按钮。

---

## 二、看板娘基础交互

### 2.1 视线跟踪

引擎持续读取鼠标在屏幕上的归一化坐标，驱动以下参数实时变化：

| 参数 | 效果 |
|:--|:--|
| `PARAM_EYE_BALL_X` / `PARAM_EYE_BALL_Y` | 眼球随鼠标方向转动 |
| `PARAM_ANGLE_X` / `PARAM_ANGLE_Y` / `PARAM_ANGLE_Z` | 头部随鼠标偏转 + 微呼吸摆动 |
| `PARAM_BODY_ANGLE_X` | 身体轻微左右倾斜 |
| `PARAM_BREATH` | 正弦波模拟呼吸起伏 |

鼠标离开页面时，所有参数归零回到正面朝向。

### 2.2 点击模型身体

点击头部 → 随机切换表情（从 `index.json` 的 expressions 列表中随机选取）。

点击身体 → 以下三种效果同时触发：
1. 从 `tap_body` 动作组随机抽取一个动画（愤怒/哭泣/害羞等）并播放
2. 从 `tapBody` 对话池随机选一条显示气泡
3. 触发彩蛋连击计数器（见第五节）

### 2.3 鼠标悬浮模型身体

悬浮身体上时触发 `live2d:hoverbody` 事件，从 `hoverBody` 对话池随机显示气泡（优先级最低，可被其他消息覆盖）。

### 2.4 控件一览

看板娘右侧有 7 个 SVG 图标按钮，hover 时渐显：

| 按钮 | 功能 | 图标 |
|:--|:--|:--|
| 一言 | 调用 hitokoto.cn API 获取一条随机句子 + 出处信息 | 💬 |
| 飞机大战 | 加载 Asteroids 小游戏，写代码累了来一把 | 🎮 |
| 切换模型 | 循环切换到下一个 MyGO 角色 | 👥 |
| 换装 | 尝试切换纹理贴图（当前模型各只有一套，会提示失败） | 👗 |
| 拍照 | 将当前 Canvas 导出为 PNG 并触发下载 | 📷 |
| 关于 | 跳转 live2d-widget GitHub 仓库 | ℹ️ |
| 退出 | 隐藏看板娘，右下角浮现小按钮可唤回 | ❌ |

退出后在右下角出现一个半隐藏的唤回按钮，点击即恢复。若关闭后 24 小时内再访问，按钮主动弹出提醒。

## 四、鼠标区域 → 表情联动

文件：`public/live2d/expression-zones.js`。核心机制：

1. 全局监听 `mousemove`
2. 用 `e.target.closest(selector)` 匹配鼠标所在的 UI 区域
3. 调用 `model.setExpression(name)` 切换模型表情
4. 每区域有冷却时间，防止频繁切换抖动
5. 离开所有区域后回退到默认 `idle` 表情

## 五、点击彩蛋系统

### 5.1 彩蛋触发

监听 `live2d:tapbody` 自定义事件（框架在点击模型身体时派发）。在 **2 秒时间窗口** 内累计连击次数，达到阈值触发彩蛋。

### 5.2 彩蛋等级

| 连击数 | 彩蛋对话 | 额外效果 |
|:--|:--|:--|
| 3 | "不要一直戳我啦…虽然不痛但是很害羞的…(\*/ω＼\*)" | — |
| 5 | "咕咕嘎嘎！！…你、你是戳戳星人吗！(╯>□<)╯" | — |
| 7 | "啊啊啊…！不要再戳了啦…代码还在等着你呢…（脸红到耳根）" | — |
| 10 🎉 | "十连击！！你赢了…我认输啦！(oT-T)ノ 咕咕嘎嘎…以后还请对我温柔一点哦…" | 模型切换为 `shame01` 害羞脸 3 秒 |
| 15+ 每 5 次 | "已经第 N 下了…求求你别戳了，再戳我就变成企鹅肉饼了！🐧"

### 5.3 彩蛋细节

- 彩蛋消息延迟 1.5 秒显示，等框架自带的 `tapBody` 消息先播完，不撞车
- 如果 2 秒内没有新的点击，计数器自动清零
- 切换模型不会重置计数器（有意为之）
- 同一次连击只触发最高等级的彩蛋（3/5/7 同时满足时只触发 7 的）

## 七、技术架构

### 7.1 文件依赖关系

```
index.html
  └── autoload.js                          ← 入口：设 base URL，异步加载以下
        ├── waifu.css                      ← 样式：定位、气泡、动画
        ├── waifu-tips.js                  ← 核心逻辑（Rollup 打包，ESM）
        │     ├── chunk/index.js           ← Cubism 2 模型类
        │     └── chunk/index2.js          ← Cubism 5 存根（未使用）
        ├── waifu-tips.json                ← 对话/季节/时段/模型配置
        ├── live2d.min.js                  ← Cubism 2 Core 引擎
        └── expression-zones.js            ← 表情联动 + 彩蛋
```

### 7.2 渲染管线

1. `autoload.js` 通过 IIFE 异步加载 CSS + JS
2. DOM 就绪后 `initWidget()` 被调用
3. `waifu-tips.js` 中的 `loadWidget()` 插入 `#waifu` DOM 结构
4. 读取 `waifu-tips.json`，注册事件监听器
5. 初始化 ModelManager → 根据配置加载第一个模型
6. 选择 Cubism 2 路径 → 动态 import `chunk/index.js` → 实例化 `Cubism2Model`
7. `Cubism2Model.init()` 获取 WebGL2 上下文 → `LAppLive2DManager.changeModelWithJSON()` → `LAppModel.loadModelSetting()` → 解析 `index.json` → 加载 `.moc` 文件 + 纹理 + 表达式 + 物理
8. `requestAnimationFrame` 驱动渲染循环

### 7.3 跨模块通信

```
autoload.js ──initWidget(config)──▶ waifu-tips.js
                                        │
                         window.__live2dModel = model    ← 暴露引用
                                        │
expression-zones.js ◀── poll + mousemove ──┘
                                           │
   model.setExpression(name)    ← 表情联动
   model.expressions[name]      ← 读取可用表情列表
   model.cubism2model.live2DMgr ← 动作/点击管理器
```

## 九、已知局限

| 局限 | 说明 |
|:--|:--|
| 选择器耦合 | 对话依赖 DOM 的 `class` 和 `[title]` 属性，前端重构时需同步更新 `waifu-tips.json` |
| 表情兼容性 | 不同 MyGO 模型的表情子集不同，已实现三级 fallback 但可能不完美 |
| 换装不可用 | 每个模型只有一套纹理，切换纹理按钮无实际效果 |
| 模型切换引用 | 切换模型后 `window.__live2dModel` 指向新实例，表情联动无缝衔接 |
| `window.Image` 劫持 | autoload.js 修改了全局 Image 构造函数的 crossOrigin，目前不影响 Vue 组件 |
