# Java 算法可视化导师 — 方案精炼版

---

## 〇、灵感来源：为什么做这个

**1. Python Tutor 启发**
Python Tutor 把代码执行过程可视化，在数据结构教学中效果极好——但 Java 版本依赖调试器接口，不够轻量。我们想用 AST 插桩这条路做出一个更干净的 Java 版本。

**2. LLM 平台的可视化趋势**
近期 ChatGPT、DeepSeek 等 LLM 平台陆续加入可视化交互模块。大厂用行动验证了一个判断：可视化能显著提升用户对知识的理解，增强平台交互性。我们把这个思路搬到代码教学上。

**3. Debug 是所有 CS 学生的痛点**
本地 IDE Debug 是每个计算机学生绕不开的坎，但 JDK 自带调试工具只有命令行文本输出，没有任何图形化辅助。我们想把 Debug 过程变成"看动画"，降低初学者的心智负担。

---

## 一、产品一句话定义

一个 Web 工具，用户输入 Java 算法代码，系统 AST 插桩真实编译执行，逐步可视化变量状态，并由大模型同步用大白话解说每一步。

---

## 二、核心策略（答辩金句）

我们用 **JavaParser AST 自动插桩 + 动态编译**，在 Web 端复现了 Python Tutor 的执行追踪能力，并融入 **LLM 实时解说**，证明了"可解释的代码可视化"可行。

## 四、技术架构

**前端 Vue 3：** Monaco Editor + 数据结构可视化画布（箭头+方框动画）+ AI 对话（SSE 流）
**后端 Spring Boot：** AST 插桩（JavaParser）+ 动态编译执行（安全沙盒）+ LLM 服务（SSE 流）

**数据流：** 代码 POST → AST 解析插桩 → 内存编译反射执行 → 每步记录变量值 → 步骤 JSON 返回 → 前端播放 + 请求 SSE 解说

---

## 四-B、前端可视化设计（核心体验）

**不只是变量卡片，是"数据结构动起来"：**

| 算法 | 可视化形式 |
|------|-----------|
| 冒泡排序 | 数组用一排方框表示，正在比较/交换的两个元素高亮，箭头标注当前指针位置 |
| 链表反转 | 节点用方框、指针用箭头连线表示，每步箭头反向转动，节点高亮当前操作对象 |
| 递归（斐波那契） | 调用栈以竖直方框堆叠展示，递归展开时方框向下生长，回溯时逐层消失 |

**实现思路：**
- 使用 **Canvas / SVG** 绘制箭头和方框（推荐 SVG，方便与 Vue 响应式绑定）
- 每一步的变量 JSON 中附带结构描述字段（如数组用 `[]` 表示，链表用 `{val, next}` 表示）
- 画布组件监听 `currentStep` 变化，驱动箭头和方框做 CSS transition / SVG animate 平滑移动
- 不影响现有的变量卡片展示——两者互补：卡片看数值，画布看结构

## 五、核心模块：AST 插桩引擎

**插桩位置：** 变量声明赋值、赋值表达式、数组元素赋值、循环变量更新、return 前插入 done()

**安全三件套：**
1. AST 扫描拒绝 `System.exit` / `Runtime.exec` 等黑名单
2. 独立线程执行，超时 5 秒强制终止
3. 仅允许 `import java.util.*`

**输出格式：**
`[{ step: 1, line: 5, variables: {i: 0, arr: [5,3,8]} }]`

---

## 六、前端播放器

**代码区：** `currentStep` 控制索引 → Monaco `deltaDecorations` 无闪烁高亮当前行

**可视化画布：** 同一 `currentStep` 驱动 SVG/Canvas 画布 → 箭头和方框随代码执行平滑移动（见四-B）

**变量卡片：** Vue 响应式渲染，值变化加 CSS 0.2s 过渡动画，只更新变了值的卡片

**AI 解说：** EventSource 接收 SSE 逐字输出 → 右侧面板打字机效果

**控件：** 自动播放 / 暂停 / 上一步 / 下一步 / 速度调节

## 十、可行性：7 个坑与对策

| # | 坑 | 一句话对策 |
|---|-----|-----------|
| 1 | 插桩后代码变形编译失败 | 用 `LexicalPreservingPrinter` 保格式；只在完整语句后插入 |
| 2 | 动态编译类加载失败 | 固定类名 `UserCode`，反射调用写单测先跑通 |
| 3 | 死循环/恶意代码 | 三层防御：AST 黑名单 → 线程 5s 超时 → 可选进程隔离 |
| 4 | 复杂类型难序列化 | MVP 只支持 `int/String/int[]`，值全存字符串 |
| 5 | LLM 胡言乱语或超时 | System Prompt + Few-shot 约束；后端 fallback；前端自动重连 |
| 6 | 高亮抖动/动画卡顿 | `deltaDecorations` 无闪烁高亮；CSS 0.2s 过渡；只更新变了值的卡片 |
| 7 | 联调字段对不上/跨域 | D2 约定 JSON 契约，双方 Mock；`@CrossOrigin`；每天 10min 站会 |

## 十三、开发平台与部署方案

### 开发阶段（D1-D12）：纯本地，零部署

整个开发期间不需要任何服务器。前后端都在各自电脑的 `localhost` 上跑：

| 端 | 运行方式 | 访问地址 |
|----|---------|---------|
| 后端 Spring Boot | IDE 直接 `mvn spring-boot:run` | `localhost:8080` |
| 前端 Vite | 终端 `npm run dev` | `localhost:5173` |

前端开发时 Vite 配置 proxy 把 `/api/*` 转发到 `localhost:8080`，绕过跨域问题。

### LLM API 调用：需要但免费

| 你需要什么 | 具体方案 |
|-----------|---------|
| API Key | 注册 DeepSeek 开放平台（新人送 500 万 tokens 免费额度，14 天开发绰绰有余）或 Qwen API（同样有免费额度） |
| 调用方式 | 后端通过 `RestTemplate` 或 OkHttp 请求 LLM API，SSE 流式转发给前端 |
| 没钱怎么办 | MVP 阶段甚至可以先用静态文本 mock 掉 LLM 调用，播放器调通了再接 API |

### 答辩现场：三种方案按需选

| 方案 | 怎么做 | 适合场景 |
|------|--------|---------|
| **A. 本地演示（推荐）** | 答辩时你的笔记本同时跑前后端，浏览器打开 `localhost:5173`，投屏展示 | 最简单，零风险，不依赖网络 |
| **B. 内网穿透** | 用 ngrok / frp 把本地 `8080` 暴露一个临时公网 URL，评委可以用自己电脑访问 | 如果有远程评委或需要手机扫码体验 |
| **C. 正式部署** | 买一台最便宜的云服务器（阿里云/腾讯云学生机，约 10 元/月），用 `mvn package` 打 jar 包部署，前端 `npm run build` 放到 Nginx 下 | 比赛结束后想长期运营或展示给更多人 |

**建议：比赛阶段用方案 A。** 你们只有 14 天，把时间花在功能上，不要分心搞运维。部署是比赛结束后的事。

## 十五、竞品分析与创新点

**与竞品对比一句话：** Python Tutor 用调试器，无 AI；JIVE/JGrasp 绑 IDE、非 Web；Copilot 只能解释静态代码，不懂运行状态。

**你的组合创新（目前无竞品完全覆盖）：**

| 能力 | Python Tutor | JIVE | Copilot | **我们** |
|------|:--:|:--:|:--:|:--:|
| Java 支持 | 部分 | ✔️ | 仅文本 | **✔️ AST插桩** |
| 逐步可视化 | ✔️ | ✔️ | ❌ | **✔️** |
| 源码级插桩 | ❌ | ❌ | ❌ | **✔️** |
| LLM 实时解说 | ❌ | ❌ | 非同步 | **✔️** |
| Web 原生无插件 | ✔️ | ❌ | ✔️ | **✔️** |
| 安全沙盒 | 受限 | ❌ | - | **✔️** |

**答辩收尾句：** 我们不是从零发明轮子，而是把 AST 源码插桩 + LLM 运行时解说这两个能力组合到了 Java Web 教学这一个空白地带——这个组合创新在本科阶段足够硬核。
