#### 省流版：一键安装并运行（windows+vscode）

  

（使用 winget）

  

```bash

# 1. 安装 Git

winget install --id Git.Git -e --source winget

  

# 2. 安装 Node.js (LTS)

winget install --id OpenJS.NodeJS.LTS -e --source winget

  

# 3. 安装 JDK 17 (Microsoft OpenJDK)

winget install --id Microsoft.OpenJDK.17 -e --source winget

  

# 4. 安装 Maven

winget install --id Apache.Maven -e --source winget

  

# 5. 关闭并重新打开 VSCode 终端（使 PATH 生效），然后验证安装

java -version

mvn -version

node -v

npm -v

git --version

  

# 6. 克隆项目仓库

git clone https://github.com/Curse-strickland/javatutor.git

cd javatutor

  

# 7. 后端启动（在新终端中执行，或使用 Ctrl+Shift+` 新开一个）

cd backend

mvn clean compile

mvn spring-boot:run

  

# 8. 前端启动（另开一个终端）（如果npm安装出现问题请在cmd中打开frontend再执行以下指令）

cd frontend

npm install

npm run dev

```

##### 克隆代码仓库

  

在 VSCode 终端中执行以下命令：

  

```bash

# 克隆项目到当前目录

git clone https://github.com/Curse-strickland/javatutor.git

  

# 进入项目文件夹

cd javatutor

  

# 后续如需更新代码，执行 git pull origin main

```

  

##### 后端部署

  

```bash

# 1. 进入后端目录

cd backend

  

# 2. 使用 Maven 编译并启动（Spring Boot 内嵌 Tomcat，默认端口 8080）

mvn clean compile

mvn spring-boot:run

  

# 或者打包为 JAR 后运行

mvn clean package -DskipTests

java -jar target/javatutor-backend-*.jar

```

  

配置说明（backend/src/main/resources/application.properties）：

  

```

# 服务器端口

server.port=8080

  

# 安全沙箱超时（毫秒）

javatutor.sandbox.timeoutMs=5000

```

  

##### 前端部署

  

```bash

# 1. 进入前端目录（新建一个终端）

cd frontend

  

# 2. 安装依赖

npm install（如果npm安装出现问题请在cmd中打开frontend再执行以下指令）

  

# 3. 启动开发服务器（默认 http://localhost:5173）

npm run dev

  

# 4. 构建生产版本（输出到 dist/ 目录）（可选）

npm run build

```

  

注意：前端开发模式下 Vite 已配置将 /api 请求代理到 http://localhost:8080（见 vite.config.js），确保后端先启动再打开前端页面。生产部署时可将 dist/ 静态文件放入后端 resources/static/ 目录，或使用 Nginx 反向代理。

  

### 5.2 使用指南

  

#### 5.2.1 基本使用流程

  

1. 打开页面：浏览器访问 http://localhost:5173

2. 编写代码：在左侧编辑器输入 Java 代码（需包含 public class 和 main 方法）

3. 运行代码：点击底部控制栏的 ▶（运行）按钮

4. 逐步查看：点击 ◀ / ▶（上一步/下一步）单步执行；拖拽进度条跳转到任意步骤；点击 ⏵（自动播放）连续播放，可调节速度

5. 查看变量与流程：右侧"变量"标签页展示当前步骤的变量卡片和内存状态；"流程"标签页实时显示 Mermaid 控制流图，当前执行步骤的节点会以黄色呼吸灯同步高亮

  

#### 5.2.2 测试模式

  

适合运行 LeetCode 风格的代码片段（无需写完整的 public class 和 main）：

  

1. 点击编辑器标题栏右侧的"测试"按钮

2. 粘贴 class Solution { public int[] bubbleSort(int[] arr) { ... } } 格式的代码

3. 在下方输入测试参数（支持 int[]、String[]、List<Integer>、ListNode、TreeNode 等类型）

4. 点击运行

5. 查看结果：控制台输出区会显示该函数的返回值（测试模式下 System.out.println 被自动替换为返回值收集）

  

#### 5.2.3 经典代码浏览器

  

点击右侧"经典"标签页，可浏览 30+ 内置算法示例：

  

- 排序：冒泡排序、快速排序、归并排序、插入排序等

- 递归：斐波那契、阶乘、汉诺塔

- 链表：反转链表、合并链表、检测环

- 树/图：DFS、BFS、二叉树遍历

- 动态规划：背包问题、最长公共子序列

  

点击任意示例即可加载到编辑器，运行查看可视化过程。

  

#### 5.2.4 AI 解说

  

AI 解说面板包含三个标签页：

  

**「解说」标签页 — 让 AI 讲解代码**

  

1. 点击控制栏右侧按钮展开 AI 面板

2. 点击 「整体解说」按钮：AI 综述整段代码的算法思路和数据结构

3. 点击 「单步解说」按钮：AI 解释当前这一步在做什么、变量为什么变化

4. 勾选 「自动」复选框：步进时自动生成解说，AI 生成中自动暂停等待，生成完毕后继续

5. 已解说的步骤会缓存，回退时从缓存恢复，不重复消耗 token

  

**「复杂度」标签页 — 自动分析算法效率**

  

- 运行代码后自动显示时间和空间复杂度（如 O(n²)、O(1)），附带文字说明

  

**「算法」标签页 — 智能识别算法类型**

  

- 自动识别代码用到的算法（排序/递归/贪心/DP 等）和数据结构（数组/链表/栈/树等），以彩色标签展示

- 点击任一标签可让 AI 针对该主题深入讲解

  

**自定义 API — 切换 AI 服务商**

  

1. 展开面板底部的「自定义 API」折叠区

2. 点击选择平台（智谱 / DeepSeek / OpenAI / Kimi / 自定义）

3. 输入对应 API Key，系统自动校验格式

4. 自定义平台需额外填写 API URL 和模型名称

5. 点击保存 — 仅本次会话生效

  

AI 解说配置：API Key 在前端界面中输入，无需在配置文件中设置。默认使用智谱 GLM-4.7 免费模型，提供免费api:2769ed6fa37a44f6b9d43f198c0f9f37.AdLpLLHPn5SqgO81(可复制）

  

#### 5.2.5 控制流图

  

1. 点击右侧「流程」标签页，查看由 Mermaid 自动生成的控制流图

2. 执行代码时，当前节点以黄色呼吸灯高亮，自动滚动跟随

3. 点击蓝色方法调用节点可钻取进入子方法的流程图，顶部面包屑可返回上层

4. 点击 🔍 按钮进入全屏模式：支持 Ctrl+滚轮缩放（25%–300%）、+/- 按钮微调、重置缩放

5. 点击 ⬇ 按钮将当前流程图下载为 SVG 文件

  

#### 5.2.6 内存可视化

  

右侧"变量"标签页展示：

  

- 栈区：函数调用栈帧，每帧显示局部变量和参数，引用变量与堆对象联动高亮

- 堆区：数组/对象/集合/Map 的字段展开，LinkedList 的节点链，对象间引用关系通过色彩标签映射

- 控制台：System.out.println() 的输出实时显示

  

#### 5.2.7 壁纸与背景

  

点击右侧卡片头部 🖼️（壁纸）按钮：

  

- 选择预设壁纸（默认网格、深邃星空、Train Girl）

- 上传自定义图片/视频/音乐作为背景

- 调节透明度（推荐 15%-30%，不干扰代码阅读）

  

#### 5.2.8 看板娘

  

右下角的高松灯（小灯）是全程陪伴的桌面助手：

  

**基础操作**

  

- 拖动：按住小灯可水平拖到屏幕任意位置，位置自动保存

- 折叠/恢复：鼠标移入看板娘时出现角标，点击角标折叠或展开

  

**交互反馈**

  

- 悬停引导：鼠标悬停在编辑器、按钮、面板等 40+ 个 UI 元素上时，小灯会弹出对应功能解说

- 触摸反应：摸头或摸身体会触发不同的害羞/抗拒台词

- 点击跟随：点击运行、步进等按钮时，小灯会给出鼓励或提醒

  

**自动事件**

  

- 时段问候：清晨/上午/中午/下午/傍晚/晚间/深夜/凌晨不同时段自动切换问候语

- 节日彩蛋：元旦、情人节、1024 程序员节等 15 个节日有专属祝福

- 控制台彩蛋：按 F12 打开浏览器控制台可见隐藏留言

- 复制提醒：在编辑器中复制代码时弹出提示

- 离开欢迎：切换到其他标签页再回来时欢迎问候

  

#### 5.2.9 文件导入与历史

  

**导入文件**

  

1. 点击编辑器标题栏「导入」按钮展开导入面板

2. 拖放 .java 文件到虚线区域，或点击选择文件

3. 支持单个文件或整个文件夹批量导入

4. 导入的代码自动加载到编辑器

  

**导入历史**

  

- 最近导入的 20 个文件自动记录在面板中

- 同名文件自动去重（替换旧记录）

- 点击历史条目可一键重新加载

- 可单独删除不需要的记录

  

#### 5.2.10 布局与视觉定制

  

**布局调整**

  

- 编辑器/面板比例：拖拽中间分割条调整左右宽度

- 控制栏位置：拖拽控制栏左侧的 6 点手柄，可将整个控制栏放到屏幕任意位置

  

**壁纸系统**

  

1. 点击右侧面板头部 🖼️ 按钮打开壁纸选择器

2. 预设壁纸：默认网格、深邃星空、Train Girl

3. 自定义背景：上传图片（JPG/PNG）或视频（MP4）作为背景，视频循环播放

4. 背景音乐：上传音频（MP3），可调节音量

5. 透明度：拖动滑块调节卡片透明度（推荐 15%–30%）

## Maven 安装简明教程（Windows）

  

前提：请先确认 JDK 17 已安装且配置正确。在命令提示符（CMD）或 PowerShell 中输入 java -version，能显示版本信息即可。

  

### 1. 下载 Maven

  

打开浏览器，访问 Maven 官网：https://maven.apache.org/download.cgi

  

找到 Files 区域，点击下载 Binary zip archive 格式的文件（例如 apache-maven-3.9.9-bin.zip）

  

### 2. 解压

  

将下载的 zip 包解压到一个没有中文、没有空格的路径，推荐：D:\apache-maven-3.9.9

  

解压完成后，记住这个路径，后面会用到

  

### 3. 配置环境变量

  

打开环境变量设置

  

右键点击"此电脑" → "属性" → "高级系统设置" → "环境变量"

  

新建 MAVEN_HOME 系统变量，在"系统变量"区域点击"新建"

  

- 变量名：MAVEN_HOME

- 变量值：填你解压的路径，例如 D:\apache-maven-3.9.9

  

点击"确定"

  

### 4. 修改 Path 变量

  

在"系统变量"中找到 Path，双击（或选中后点击"编辑"）

  

点击"新建"，输入 %MAVEN_HOME%\bin

  

点击"确定"关闭所有窗口

  

### 5. 验证安装

  

重新打开命令提示符（CMD）或 PowerShell（必须新开窗口，环境变量才会生效）输入 mvn -v 并回车，如果看到 Apache Maven 3.9.9 和 Java 版本信息，说明安装成功
